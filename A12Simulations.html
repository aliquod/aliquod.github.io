Monte Carlo Integration

Rejection Sampling

  • Suppose we want to sample X∼pX \sim pX∼p (pdf), and have another variable Y∼q≠0Y \sim q \ne 0Y∼q=0 that is simpler to simulate.
  • Instead of manipulating YYY to behave like XXX (like in transformation sampling), rejection sampling “rejects” values of YYY to make the remainder representative of a sample from X∼pX \sim pX∼p.
  • Y∼qY \sim qY∼q is then called the proposal (distribution), while X∼pX \sim pX∼p is the target (distribution).

How it works

  • To make the proposal look like the target, we need to rescale its distribution: at Y=yY=yY=y, it should be rescaled by p(y)/q(y)p(y) / q(y)p(y)/q(y).
  • This is done by randomly accepting the proposal with some probability L∝p(y)/q(y)L \propto p(y) / q(y)L∝p(y)/q(y) . However, for LLL to be a valid probability, it needs to be in [0,1][0,1][0,1]. Hence find some M:p/q≤MM: p / q \le MM:p/q≤M everywhere, then define the threshold threshold L(y)=p(y)q(y)1M∈[0,1]\text{threshold } L(y)=\frac{p(y)}{q(y)} \frac{1}{M} \in [0,1]threshold L(y)=q(y)p(y)​M1​∈[0,1]
  • To accept the proposal with this probability, we then simulate U∼U[0,1]U \sim U[0,1]U∼U[0,1] and accept if and only if U≤LU \le LU≤L.

The Algorithm

  • The rejection sampling algorithm simulates X∼pX \sim pX∼p by:

    • (1) sample Y∼qY \sim qY∼q, giving Y=yY=yY=y.
    • (2) simulate U∼U[0,1]U \sim U [0,1]U∼U[0,1].
    • (3) if U≤p(y)/Mq(y)U \le p(y) / Mq(y)U≤p(y)/Mq(y), accept and return X=yX=yX=y. Otherwise reject and return to (1).

  • Importance sampling estimates θ=E[ϕ(X)]\theta=\mathbb{E}[\phi(X)]θ=E[ϕ(X)] for some random variable X∼pX \sim pX∼p using a proposal Y∼qY \sim qY∼q, similar to rejection sampling.

Markov Chain Monte Carlo

  • MCMC estimate θ=E[ϕ(X)]\theta = \mathbb{E}[\phi(X)]θ=E[ϕ(X)] by simulating X1,X2,⋯∼Markov(P,λ0)X_{1},X_{2},\dots \sim \text{Markov}(P,\lambda_{0})X1​,X2​,⋯∼Markov(P,λ0​) as samples of X∼pX \sim pX∼p, defined over a discrete state space III.
    • PPP is the transition matrix, chosen so that the samples converge to ppp.

Reversible Markov Chains

Big idea: reversible chains satisfy the detailed balance equations. Given a distribution ppp, those equations help find a transition matrix PPP that has ppp as a stationary distribution.

  • Suppose X=(X1,X2,… )∼Markov(P)\mathbf{X}=(X_{1},X_{2},\dots) \sim \text{Markov}(P)X=(X1​,X2​,…)∼Markov(P) has stationary distribution π\piπ.

  • Its reversed chain is Y=(Y1,…,Yn):Yi=Xn−i+1\mathbf{Y}=(Y_{1},\dots,Y_{n}):Y_{i}=X_{n-i+1}Y=(Y1​,…,Yn​):Yi​=Xn−i+1​. If X\mathbf{X}X started from its stationary distribution π\piπ, then Y∼Markov(P~)\mathbf{Y} \sim\text{Markov}(\tilde{P})Y∼Markov(P~) is also a Markov chain.

    • That is, Y\mathbf{Y}Y is X1,…,XnX_{1},\dots,X_{n}X1​,…,Xn​ set in reverse order.

  • The chain X∼Markov(P,π)\mathbf{X} \sim\text{Markov}(P,\pi)X∼Markov(P,π) with a reverse Y\mathbf{Y}Y is reversible (wrt. π\piπ) if for any x1,…,xnx_{1},\dots,x_{n}x1​,…,xn​ from the sample space, P[X1=x1,…,Xn=xn]=P[Y1=x1,…,Yn=xn]\mathbb{P}[X_{1}=x_{1},\dots,X_{n}=x_{n}]=\mathbb{P}[Y_{1}=x_{1},\dots,Y_{n}=x_{n}]P[X1​=x1​,…,Xn​=xn​]=P[Y1​=x1​,…,Yn​=xn​]That is, the chain X\mathbf{X}X and its reverse Y\mathbf{Y}Y are indistinguishable if they start from the same distribution.

Metropolis-Hastings Algorithm

Big idea: Metropolis-Hastings indirectly simulates the suitable Markov chain that converges to the target distribution.

  • Metropolis-Hastings indirectly simulates the transition Xk→Xk+1X_{k}\to X_{k+1}Xk​→Xk+1​ by rejection sampling: a proposal distribution qqq gives the next state, which is accepted/rejected: [Xk=i]undefinedby qproposal[Xk+1=j ?]→{accept→[Xk+1=j]reject  →[Xk+1=Xk=i][X_{k}=i] \xrightarrow[\text{by }q]{\text{proposal}}[X_{k+1}=j\,?]\to\begin{cases} \text{accept}\to [X_{k+1}=j] \\ \text{reject }\,\to [X_{k+1}=X_{k}=i] \end{cases}[Xk​=i]proposalby q​[Xk+1​=j?]→{accept→[Xk+1​=j]reject →[Xk+1​=Xk​=i]​where the current state is carried over if the proposal is rejected.

p <- partial(dchisq, df=10)  
q <- function(xt){runif(1, min=xt-1, max=xt+1)}  
q_density <- function(from, to){1/2}

accept <- function(prev, proposed){
  U <- runif(1)
  fraction1 <- p(proposed)/p(prev)
  fraction2 <- q_density(proposed, prev) / q_density(prev, proposed)
  accept_prob <- min(1, fraction1 * fraction2)
  return(U < accept_prob)
}


n <- 100000
previous_state <- 1

for (i in 1:n){
  proposal <- q(previous_state)
  accepted <- accept(previous_state, proposal)
  if (accepted){
    previous_state <- proposal
    samples[i] <- proposal
  } else {
    samples[i] <- previous_state
  }
}
